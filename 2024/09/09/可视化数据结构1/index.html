可视化数据结构1

1. 饼状图

要求的数据结构:

如何转换:

  • 从原始数据：聚合数据，计算每个类别的总值或百分比。
  • 示例: 将每个类别的总金额（如地区或产品）转换为 name 和 value 对应的结构。

2. 散点图

要求的数据结构:

如何转换:

  • 从原始数据：需要将数据点的横坐标和纵坐标组织成二维数组。
  • 示例: 将时间序列或其他类型的分布数据转换成 [x, y] 格式。

3. 柱状图

要求的数据结构:

如何转换:

  • 从原始数据：将类别和对应的数值分开存储，并确保它们的顺序一致。
  • 示例: 将不同类别的总金额和名称分开，分别转换成 categories 和 values 数组。

4. 线形图

要求的数据结构:

如何转换:

  • 从原始数据：将时间序列数据或顺序数据转化为按时间排序的键值对数组。
  • 示例: 将时间点和对应的值组织成 { "time": ..., "value": ... } 形式的数组。

5. 热力图

要求的数据结构:

如何转换:

  • 从原始数据：将坐标和强度值整理成 { "x": ..., "y": ..., "value": ... } 格式。
  • 示例: 将数据点的坐标和强度（如密度）转化为热力图需要的数据结构。

转化流程总结

  1. 数据汇总: 聚合和总结原始数据，计算每个类别、时间点、或坐标点的值。
  2. 数据格式化: 将数据转换成图表库要求的格式。
  3. 数据映射: 依据图表类型的需求，将数据映射到正确的属性上，例如 name, value, x, y 等。
